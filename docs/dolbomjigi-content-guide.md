## 1. 톤·스타일

### 1.1 기본 톤

- **존중**: 어르신·고령자 도메인이므로 비하·동정·과장 금지
- **따뜻함**: 자녀군 페르소나 페이지(T3)는 공감 어조
- **전문성**: 사회복지사·기관 페이지(T3·T6)는 실무 톤
- **객관성**: 정부 사업 페이지(T4)는 사실 인용 중심

### 1.2 호칭·표현 표준

| 권장 (✅) | 비권장 (⚠️) | 금지 (❌) | 메모 |
|---|---|---|---|
| 어르신 | 노인 | 늙은이 | 본문 기본 호칭은 "어르신". 통계·객관 인용은 "노인"·"고령자" 허용 |
| 사용자, 이용자 | 대상자 | 환자 | "환자"는 의료기기 정책 충돌 |
| 부모님 | 양친 | 영감, 할매 | 자녀 페르소나 페이지에서 |
| 자녀 | 보호자 | 자식 | 자녀군 페르소나 |
| 사회복지사, 생활지원사 | 봉사자 | 도우미 | 정확한 직책 사용 |
| 도움, 지원, 보조 | 케어 | 케어링 | 한국어 우선 |
| 안내, 알림 | 푸시 | 알람 | 알람은 시계 의미 |

### 1.3 문장 길이

- 한 문장 최대 80자 권장 (LLM·검색 모두 짧은 문장 선호)
- 한 단락 100~250자 (50~150 단어 청크 원칙)
- 한 페이지 평균 단락 6~12개

### 1.4 수치 표기

- 본문 가독성: "1만 대", "198만원", "약 7,000대"
- 표·데이터: 정확값 "10,000대", "1,980,000원", "7,000대"
- 통계 인용: 정확값 + 출처

## 2. 페이지 구조 공통 규칙

### 2.1 헤딩 구조

```
<h1>페이지 제목 (1개만)</h1>
  <h2>대섹션</h2>
    <h3>소섹션</h3>
```

- `<h1>`은 반드시 1개
- `<h2>` 3~7개 권장
- `<h3>` 깊이는 필요할 때만
- 헤딩에 핵심 키워드 포함 (자연스럽게)

### 2.2 250자 답변 블록 (AEO 핵심)

모든 페이지는 `<h1>` 바로 아래 `<AnswerBlock>` 컴포넌트로 250자 이내 핵심 답변을 노출:

```markdown
<AnswerBlock>
효돌은 ㈜효돌이 개발한 7살 손주 모습의 AI 돌봄로봇 인형입니다. 
양방향 음성 대화, 복약·식사 알림, 움직임 감지 기능을 갖췄으며 
2025년 기준 전국에 1만 대 이상이 보급됐습니다. 
개인 구매는 ㈜효돌 공식몰에서 가능하며, 지자체 보급 대상자는 무상 또는 저비용으로 신청할 수 있습니다.
</AnswerBlock>
```

**규칙**:
- 페이지 핵심 질문에 한 문단으로 답
- 250자 이내 (한글 기준, 공백 포함)
- 핵심 키워드 자연스럽게 포함
- 숫자·고유명사·연도 명시 → LLM 인용 매력 ↑

### 2.3 본문 청크 구조

- 한 단락 = 한 주제 = 100~250자
- 단락 사이 빈 줄
- 리스트는 3~7항목 (너무 길면 분할)
- 표는 최대 8행 권장

### 2.4 내부 링크 (Cross-Tier)

각 페이지 본문에 다른 Tier 페이지로의 내부 링크 **최소 3개**:

| 페이지 | 권장 cross-link |
|---|---|
| T1 제품 | T2 비교, T3 페르소나, T4 지원사업 |
| T2 비교 | T1 제품 ×2, T3 페르소나 |
| T3 페르소나 | T1 추천 제품, T4 지원사업 |
| T4 지원사업 | T1 적용 제품, 인근 지역 T4 |
| T5 정보 | T1, T3 |
| T6 B2B | T1, T4 |

### 2.5 출처·인용 블록 (필수)

모든 페이지 하단에 `<Sources>` 섹션:

```markdown
## 출처

- [공공누리 제1유형] 보건복지부 보도자료 "민-관-수요자가 함께 돌봄로봇 개발과 활용을 위해 논의", 2024-10-28, https://...
- ㈜효돌 공식 홈페이지 제품 정보, 2025-05-13 확인, https://hyodol.com/
- 보건복지부 국립재활원 중앙보조기기센터 "어르신들의 건강과 안전을 위한 스마트 AI인형 효돌", https://www.knat.go.kr/...
```

### 2.6 메타데이터


## 3. T1 제품 페이지 작성 가이드

### 3.1 분량

- 본문 1,500~2,500자
- 섹션 7~10개
- 이미지 3~6개 (제조사 공식 + 사용 환경)

### 3.2 섹션 구조

```markdown
# {제품명} — 가격·기능·신청 가이드

<AnswerBlock>250자 핵심 답변</AnswerBlock>

## {제품명}은 어떤 돌봄로봇인가
3~5문장. 제조사·출시연도·카테고리·핵심 차별점.

## 주요 기능 5가지
- 기능 1: 1~2문장
- 기능 2: 1~2문장
- 기능 3: 1~2문장
- 기능 4: 1~2문장
- 기능 5: 1~2문장

## 가격·구독·대여 옵션
표 형식. 구매가, 월 구독, 대여, A/S 비용.

## 어떤 사용자에게 적합한가
3~4 페르소나별 적합도. T3 페이지 cross-link.

## 지자체 보급 현황
보급 사례·연도·대수. T4 페이지 cross-link.

## 비교: {제품명} vs 다른 돌봄로봇
2~3개 추천 비교. T2 페이지 cross-link.

## 자주 묻는 질문
5~8개 FAQ.

## 출처
공식·정부·언론 출처 표시.

## 작성자·감수자
EEAT 블록.
```

### 3.3 작성 룰

**좋은 예**:
> 효돌은 ㈜효돌이 개발한 인형형 AI 돌봄로봇으로, 7살 손주의 모습으로 디자인됐습니다. 머리를 쓰다듬거나 등을 토닥이면 음성으로 반응하며, 보호자가 모바일 앱으로 녹음한 안부 메시지를 정해진 시간에 재생합니다. 보건복지부 국립재활원 중앙보조기기센터는 효돌을 "치매 예방 프로그램"으로 소개했지만, 본 사이트에서는 의료 기기 광고 정책에 따라 "인지 건강 지원" 기능으로 설명합니다.

(객관적 정보 + 인용 + 정책 안전 표현 + EEAT 친화)

**나쁜 예**:
> 효돌은 정말 대단한 로봇입니다! 부모님 치매를 예방해주고 치료에 도움이 됩니다. 의사들도 추천하는 효도 1등 선물!

(과장 + 의료 표현 금지 위반 + 출처 없음 + 광고성)

### 3.4 가격·옵션 표 형식

```markdown
| 옵션 | 가격 | A/S | 비고 |
|---|---|---|---|
| 일반 구매 | 198만원 | 1년 무상 | 약정 없음 |
| 월 구독 | 4만 9,800원 | 포함 | 24개월 약정 |
| 지자체 보급 | 무상 | 포함 | 자격 요건 충족 시 |
| 개인 대여 | 월 2만 2,000원 | 포함 | 약정 기간 협의 |
```

가격은 **반드시 출처와 함께**, 확인일 표시. 가격 변동 가능성 명시.

### 3.5 FAQ 작성 룰

각 FAQ 답변 100~250자. 답변 자체로 완결되도록 (LLM 인용 친화).

```markdown
**Q. 효돌은 와이파이가 필요한가요?**
A. 네, 효돌은 LTE/Wi-Fi 통신으로 보호자 앱과 연동되며, 음성 메시지 수신과 응급 상황 알림에 통신망이 필요합니다. ㈜효돌은 통신비를 포함한 월 약정 요금제를 별도로 운영합니다.

**Q. 보급사업으로 받은 효돌도 개인 구매와 기능이 같나요?**
A. 기본 기능(음성 인터랙션, 알림, 응급 호출)은 동일합니다. 다만 보급사업에는 지자체 응급관제 시스템 연동이 추가되는 경우가 많고, 개인 구매는 보호자 단독 관제로 운영됩니다.
```

```markdown

## 한 줄 결론

## 핵심 차이 한눈에 보기
| 차원 | {A} | {B} |
|---|---|---|
| ... | ... | ... |

## 페르소나별 추천
- [페르소나 1]: [추천 제품] + 이유
- [페르소나 2]: [추천 제품] + 이유

## 가격·운영비 비교
3년 운영비 시뮬레이션 권장.

## 공식 평가·인용
정부·언론·연구 인용 자료.

## 결론

## 출처
```

### 4.3 비교 차원 표준 풀

다음 중 페이지에 적합한 5~10개 선정:

- 형태 (인형형/로봇형/스크린형)
- 주요 기능 수
- 음성 대화 방식 (양방향/단방향)
- 응급 관제 연동 여부
- 보호자 앱 기능
- 가격 (구매/구독/대여)
- 통신 방식 (LTE/Wi-Fi)
- A/S 정책
- 지자체 보급 여부
- 누적 보급 대수
- 출시 연도
- 제조사 본사 위치
- 사용 가능 언어

### 4.4 주의사항

- **승자 선언 X**. "A가 더 좋다" 식 표현 금지. "X 상황에는 A, Y 상황에는 B" 식 분기 추천만.
- 제조사 비방·과장 X
- 사용자 후기 인용 시 출처 명시 + 인용 형태 (직접 재구성 X)

각 T3 페이지 첫 섹션에서 페르소나를 구체적으로 정의:

```markdown

- 상황: 80대 어머니가 시골에 혼자 거주
- 고민: 안부 확인이 어렵고, 외로움이 걱정됨
- 예산: 100~200만원대
```

### 5.3 섹션 구조

```markdown
# {페르소나 시나리오 제목}

<AnswerBlock>250자 핵심 답변</AnswerBlock>

## 이 가이드는 누구를 위한 글인가
[페르소나 정의 4~5줄]

## 이 상황에서 돌봄로봇이 도움이 되는 이유
3~5문단. 객관적 근거 + 학술/정부 인용.

## 핵심 기능 우선순위
이 페르소나에게 필요한 기능 5~7개 우선순위.

## 추천 제품 TOP 3
- 1위: {제품} — 이유, 가격, 신청법 → T1
- 2위: {제품} — 이유, 가격, 신청법 → T1
- 3위: {제품} — 이유, 가격, 신청법 → T1

## 지자체·정부 지원 받을 수 있는가
T4 cross-link. 자격 요건 요약.

## 도입 전 체크리스트
5~8개 체크 항목.

## 자주 묻는 질문
5~7개.

## 작성자·감수자
[필수] 감수자 자격증·경력 명시.
```

## 6. T4 지원사업 페이지 작성 가이드

- 본문 800~1,500자 (인용 형태 중심)

### 6.2 섹션 구조

### 9.3 코드 차원 강제

```typescript
// src/lib/content-linter.ts
export const FORBIDDEN_MEDICAL = [
  '치매 예방', '치매예방', '치료', '처방', '의료기기',
  '환자', '진단', '효능', '약효', '처치', '부작용',
  '임상 결과', '의사 추천', '만병통치', 'FDA 승인',
];

export const FORBIDDEN_PROMOTIONAL = [
  '최고의', '1등 추천', '무조건 추천', '강력 추천',
  '후회 없는', '단 한 번뿐', '100% 효과', '100% 안전',
];

export function lintContent(text: string): LintResult {
  const medical = FORBIDDEN_MEDICAL.filter(t => text.includes(t));
  const promo = FORBIDDEN_PROMOTIONAL.filter(t => text.includes(t));
  return {
    ok: medical.length === 0 && promo.length === 0,
    violations: { medical, promotional: promo },
  };
}
```

CI에서 모든 콘텐츠 파일·DB row 게시 전 lint 통과 강제.

**예외 화이트리스트**: 인용 블록(`<blockquote>` 내) 안의 정부·언론 원문은 lint 제외. 인용 표시 명확히 (출처 URL 필수).

## 10. 출처·인용·라이선스 가이드

### 10.1 출처 표기 형식

**정부 자료**:
```markdown
- [공공누리 제1유형] 보건복지부, "민-관-수요자가 함께 돌봄로봇 개발과 활용을 위해 논의", 보도자료, 2024-10-28, https://www.mohw.go.kr/...
```

**학술 자료**:
```markdown
- [연구] 저자명, "논문 제목", 저널명 권(호), 페이지, 연도, DOI: 10.xxxx/xxxxx
```

**언론 인용**:
```markdown
- [인용] 매체명, "기사 제목", 기자명, 게재일, URL
```

**기업 공식 자료**:
```markdown
- ㈜효돌 공식 홈페이지 제품 정보, 2026-05-13 확인, https://hyodol.com/
```

### 10.2 공공누리 라이선스 처리

- **제1유형 (출처표시)**: 자유 이용 가능. 출처 명시 필수.
- **제2유형 (출처표시 + 상업적 이용 금지)**: AdSense 사이트에서 사용 불가. 회피.
- **제3유형 (출처표시 + 변경 금지)**: 변경 없이 인용만. 본문 인용 형태로.
- **제4유형 (제3 + 상업적 이용 금지)**: AdSense 사이트 사용 불가. 회피.

**확인 절차**: 정부 자료 사용 전 원문 페이지에서 공공누리 마크 확인. 마크 없으면 사용 금지.

### 10.3 인용 형식 (저작권 안전)

- **단편 인용 (1문장 이내, 따옴표 + 출처)**: OK
- **장문 의역 (50자 이상 핵심 표현 그대로 복제)**: 저작권 위반 위험. 의역 후 출처 표시
- **연속 단편 인용 묶음 (한 출처에서 여러 단편)**: 사실상 복제. 의역 형태로

### 10.4 이미지 출처

- 제조사 공식 이미지: 자료실·공식 미디어 키트에서 다운로드 가능 표시 있는 것만
- 정부 보도자료 이미지: 공공누리 라이선스 확인 후 사용
- 자체 촬영: 우선 (Phase 2부터 권장)
- 스톡 이미지: Unsplash·Pexels 무료 라이선스만

모든 이미지에 alt 텍스트 + 출처 표기 (`<figcaption>` 또는 메타).

- 가격·신청 정보는 직접 검증 (확인일 표시)

## 14. 검수 워크플로우

```
[1] 작성자 초안 작성
   ↓
[2] 자체 검수 (체크리스트 17.0)
   ↓
[3] Content Linter 통과
   ↓
[4] T3·T6: 외부 감수자 검수 (필수)
   T4: Review queue 통과 (필수)
   T1·T2·T5: 편집팀 내부 검수
   ↓
[5] JSON-LD 검증 (Google Rich Results Test)
   ↓
[6] 게시 → IndexNow 즉시 ping → GSC·Naver Search Advisor 제출
   ↓
[7] 30일 후 트래픽·인용 데이터 모니터링
   ↓
[8] 분기별 갱신 (가격·신청 정보 변경 시)
```

- [ ] 내부 cross-link 최소 3개

- [ ] Content Linter 통과 (의료·광고 표현 금지)

- [ ] 공공누리 라이선스 확인·표시 (T4)
- [ ] 인용 형태 (단편 < 1문장, 의역 우선)
- [ ] 가격·신청 정보 확인일 표시

### 16.3 SEO·AEO

- [ ] `<title>` 50~60자, 메인 키워드 포함
- [ ] `<meta description>` 120~155자
- [ ] OG·Twitter 카드 메타
- [ ] 250자 답변 블록 핵심 정보 자체 완결

## 부록 A: 좋은 예 / 나쁜 예 샘플

### 샘플 1: T1 효돌 페이지 첫 문단

**❌ 나쁜 예**:
> 효돌은 정말 대단한 돌봄로봇입니다! 부모님 치매를 예방하고 외로움을 100% 해소해드립니다. 의사들도 추천하는 효도 1등 선물이며, 지금 바로 신청하세요!

(과장 + 의료 표현 + 광고성 + 출처 없음 + 행동 압박)

**✅ 좋은 예**:
> 효돌은 ㈜효돌(2009년 설립)이 개발한 인형형 AI 돌봄로봇으로, 7살 손주의 모습으로 디자인됐습니다. 양방향 음성 대화, 복약·식사 알림, 움직임 감지를 통한 안전 관리 기능을 제공하며, 보건복지부 국립재활원 중앙보조기기센터에 인지 건강 지원 보조기기로 등재돼 있습니다. 2025년 기준 전국에 1만 대 이상 보급됐으며, ㈜효돌 공식몰에서 개인 구매·대여가 가능합니다.

(객관적 정보 + 출처 + 안전 표현 + EEAT 신호)

### 샘플 2: T4 지자체 페이지 갱신 표시

**❌ 나쁜 예**:

(모호함 + 출처 없음 + 시점 불명)

>
> **출처 및 라이선스**: 본 페이지는 서울시 보도자료를 공공누리 제1유형(출처표시)에 따라 인용·재구성하여 작성됐습니다. 원문 확인일: 2026-05-17.

(출처 + 라이선스 + 확인일 + 정확한 수치)

### 샘플 3: T3 페르소나 페이지 마무리

**❌ 나쁜 예**:

(과장 + 광고성 + 감정 압박)

**✅ 좋은 예**:
